Discuss the scope of var, let, and const

var : "var" keyword is used to declare variables. The keyword 'var' has some different characteristics and behavior which differentiate itself from other keywords like "let" and "const".

Well, "var" is called Function scope. Variables declared with "var" are Function-scoped.They are only accessible within the function in which they are defined.

"var" is also called no-block -scope.It is accessible from outside of any function.

Variables declared with "var" outside of any function become global variables and can be accessed throughout the entire script. variables are declared with "var" can be redeclared and reassigned with the same scope without errors.

let : The "let" keyword is used for variable declaration in JavaScript, and it has some important characteristics and behaviors. "let" is block-scoped variable keyword. This makes specially useful for preventing variable leakage and creating variables that are only accessible within a specific block of code.
variables declared with "let" cannot be redeclared in the same block or scope. While you cannot redeclare that variables in the same scope, you can still update their values.
variables declared with "let" are Function-scoped, which means they're only accessible within the function in which they're defined. "let" commonly used in modern javascript development because it provides blockscope elements, hosting related issues and encourages better variables scoping practice.

const : The "const" keyword in javascript is used for variable declaration, butit has some unique characteristics and behaviors compared.to "let" and "var".
Variables declared with "const" are used to store values that should not be reassigned after their initial assignment. In other words, they're constant.
Like that, "const" is block-scope. it is confined to the block within Curly brackets. Variable declared with "const" are not hoisted to the top of their containing block, similar to let you cannot access them before their declaration.
With "const" neither you can redeclare a variable nor reassign a variable. "const" is often used for values that are meant to remain constant throughout the program's execution, such as mathematical constants or configuration settings. It is also used when you want to ensure that a variable is not accidentally reassigned.However, it is important to remember that for objects and arrays, declare with "const" you can still modify their contents.
